# NPC Secret Revelation — Design Spec
**Date:** 2026-04-19

## Problem

Every NPC system prompt tells the character their secret verbatim. The only instruction governing revelation is "be evasive but not impossibly so." The existing code triggers secret disclosure mechanically when affection reaches "devoted" — a single path with no nuance. There is no pressure path, no guilt path, no distinction between a character who cracks under a hard stare and one who needs to be slowly trusted.

## Goal

Make secret revelation feel like a consequence of how the player actually played the interrogation. A character with low pressure tolerance breaks when cornered with evidence. A high-empathy character who feels guilty confesses when the player mentions the victim's children. A character who responds to warmth opens up over multiple conversations as affection builds. The existing romance path is preserved as one of several routes.

---

## Section 1: NPC Psychological Profile

Six new fields added to `REQUIRED_SUSPECT_FIELDS` in `generator.py` and stored on the `npcs` table:

| Field | Type | Description |
|-------|------|-------------|
| `pressure_tolerance` | int (1–10) | Resistance to being pushed. 1 = cracks immediately, 10 = barely flinches |
| `kindness_weight` | int (1–10) | How much sympathy and warmth moves them |
| `empathy` | int (1–10) | How much their guilt responds to emotional appeals about victims and consequences |
| `starting_guilt` | int (0–10) | Initial guilt level at case start |
| `revelation_style` | string | `"staged"` or `"sudden"` |
| `revelation_stages` | int (2–5) | Number of stages. Only meaningful for staged NPCs; sudden NPCs ignore this field |

These are generated by the LLM alongside the existing personality, speech style, and alignment fields. The generation prompt instructs the LLM to assign values consistent with the character's role, backstory, and alignment — a remorseful witness should have high empathy and high starting guilt; a calculating killer should have high pressure tolerance and low empathy.

---

## Section 2: Dynamic State Tracking

Three new columns on `npc_relationships` (the table already tracking affection and milestone flags):

| Column | Type | Initial Value | Description |
|--------|------|---------------|-------------|
| `guilt` | integer (0–100) | `starting_guilt * 10` | Current guilt level. Never decays |
| `pressure_score` | integer (0–100) | 0 | Accumulated pressure. Decays 5 per turn when no pressure events fire |
| `revelation_stage` | integer | 0 | Stages revealed so far. 0 = nothing revealed yet |

The existing `secret_revealed` flag is unchanged. When the final revelation stage fires (or a sudden revelation fires), `secret_revealed` is set to `True` — both the affection path and the psychological path converge on the same flag, preventing double-revelation.

---

## Section 3: Event Classification

After each NPC exchange, a lightweight Haiku call classifies what just happened. Input: the player's message and the NPC's response. Output: five booleans.

| Event | Description |
|-------|-------------|
| `pressure_applied` | Player pushed on the same topic or confronted with contradictory evidence |
| `threat_made` | Player threatened the NPC explicitly |
| `kindness_shown` | Player was sympathetic, warm, or supportive |
| `guilt_trigger` | Player referenced victim suffering, consequences, people left behind |
| `evidence_confronted` | Player cited a specific contradiction in the NPC's story |

### State Updates

```
pressure_applied    → pressure_score += (11 - pressure_tolerance) * 5
threat_made         → pressure_score += (11 - pressure_tolerance) * 10
evidence_confronted → pressure_score += (11 - pressure_tolerance) * 8
kindness_shown      → guilt += empathy * 2
guilt_trigger       → guilt += empathy * 4
```

Pressure decays 5 per turn where no pressure events fire. Guilt never decays.

Both scores are clamped to [0, 100].

---

## Section 4: Revelation Threshold & Trigger

### Combined Score

```
combined = pressure_score + guilt + (affection * kindness_weight / 10)
```

Affection contributes more to characters who respond to warmth (`kindness_weight` high), less to cold or pressure-driven ones.

### Sudden NPCs

One threshold at 100. When `combined >= 100`, fire the revelation call — full secret comes out.

### Staged NPCs

Thresholds are evenly spaced across `revelation_stages`. Examples:

| Stages | Thresholds |
|--------|------------|
| 2 | 60, 100 |
| 3 | 50, 75, 100 |
| 4 | 40, 60, 80, 100 |
| 5 | 35, 55, 70, 85, 100 |

Each threshold crossed fires one revelation call for that stage number.

### Guilt Override

If `guilt >= 90`, revelation fires regardless of the combined score threshold. This is the "couldn't live with it" path — a character who confesses without being broken. Guilt override respects staging: it advances one stage per turn it fires, not all at once.

### Revelation Call

A separate `npc.speak()` call fires after the normal exchange (same pattern as `_check_npc_romance_milestone`). The prompt:

**Staged:**
```
[You are about to reveal something you have been hiding.
This is stage {n} of {total} — reveal approximately 1/{total} of your secret.
Do not reveal more than this stage calls for.
What broke you open just now: {which events fired this turn}.
Stay fully in character. Do not announce that you are confessing or making a speech about it.
Speak naturally, as the moment demands.]
```

**Sudden:**
```
[You have reached your breaking point. Tell the detective your secret — all of it.
What broke you: {which events fired this turn}.
Stay fully in character. No speeches. Speak the way this person actually speaks.]
```

The `record=False` flag is set so the revelation is not stored in conversation history as a normal exchange — it is a milestone moment, not a turn.

## Out of Scope

- Player-facing display of NPC psychological state
- Psychological traits for fixed NPCs (Lou, the DA) — they use defaults
- Pressure decay rate tuning (hardcoded at 5 per turn; can be adjusted later)
- Alignment affecting revelation thresholds (alignment already affects NPC openness via the existing prefix system)
