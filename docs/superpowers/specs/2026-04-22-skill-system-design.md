# Skill System — Design Spec
**Date:** 2026-04-22
**Status:** Approved

---

## Overview

A skill system that grows from how the player actually plays — no menus to navigate, no points to assign. Roots are determined by the alignment quiz. Specializations are generated by the LLM from the player's action history. The partner has her own complementary skill trees and earns XP through interjections in NPC conversations. Growth is mostly invisible, surfacing only through ambient partner lines and the `/me` command.

---

## Section 1: Roots & Trees

Four root skills, each tied to one alignment pole:

| Root | Axis |
|------|------|
| **Authority** | Lawful |
| **Streetwise** | Chaotic |
| **Empathy** | Good |
| **Cunning** | Evil |

The alignment quiz places the player on two axes (`law_chaos`, `good_evil`). The player starts with whichever two roots their quiz alignment leans toward, at level 1. The partner starts with the complementary two at level 1. True Neutral players start at level 1 in all four roots with no dominant pair.

**Specializations** are not predefined. Every time a root reaches a level divisible by 3, the LLM generates a new specialization for that root — named and described in 1-2 sentences in the world's voice, based on the player's recent `skill_events` for that root. A specialization is a narrative capability: it gets injected into relevant conversation and action contexts. There is no cap on how many specializations a root can accumulate.

Partner specializations are generated separately from the partner's own interjection history, diverging from the player's over time.

---

## Section 2: Growth Mechanics

Each root has an XP track. XP is awarded after conversations via the existing post-conversation summary LLM call — a new `xp_awards` field is added to the JSON response, listing which roots earned XP and how much, based on what actually happened in the conversation.

**Alignment weighting:** Actions aligned with the player's current alignment direction award full XP to the corresponding root. Misaligned actions award reduced XP. The further the alignment leans, the stronger the bonus for aligned actions — a smooth gradient, not a cliff. Alignment axes are unbounded; the longer a player acts consistently, the deeper the lean and the stronger the growth signal.

**XP tiers:**
- Normal aligned action: base XP
- Lucky unskilled success (see Section 3): bonus XP — the attempt cost something and paid off
- Story milestones (suspect breaks, dark past resolves, corrupt official exposed): large XP boost to the most-used root in that case
- Skilled success: standard base XP (you're doing what you're built for)

Root level thresholds are fixed (e.g., 100 XP per level). When a root reaches a multiple of 3, specialization generation fires.

---

## Section 3: Backfire & Success Probability

The player can always attempt anything. Skill level affects the probability distribution of outcomes — it never locks a door.

**Three-outcome probability space:**

| Situation | Success chance | Backfire chance | Lucky critical |
|-----------|---------------|-----------------|----------------|
| Skilled attempt | High | Low | Rare |
| Unskilled attempt | Moderate | Meaningful | Small but real |
| Wildly outskilled | Low | High | Near zero but nonzero |

**Implementation:** Before the NPC responds, relevant skill levels are checked against the difficulty of the attempt (a hardened mob enforcer is harder to intimidate than a nervous clerk). A probability roll determines the outcome bucket. The NPC prompt is injected with the outcome: success, backfire, or lucky break.

**Narrative shape of outcomes:**
- *Backfire:* suspect goes cold, sees through the bluff, warns others, trusts less for the rest of the case
- *Unskilled success:* feels desperate, surprising — narratively distinct from a skilled success, may cost social capital elsewhere
- *Lucky critical:* rare, high-reward, earns bonus XP

---

## Section 4: Partner Complementarity & Partner Turn

The partner's roots are the complementary two to the player's. Her specializations generate from her own interjection history. Her skill trees grow in parallel using the same XP mechanics and alignment weighting applied to her alignment.

**Partner turn in the conversation loop:** After the NPC responds to the player, a lightweight context check runs. If any of the following are true, the partner interjects:

- The NPC said something the partner's skill set is positioned to exploit
- The player's approach is visibly failing (backfire in progress)
- A clue thread appears in the NPC's response that the partner's specialization would catch

The interjection is an LLM call with full conversation context, the partner's skill profile, and an instruction to apply pressure, provide cover, redirect the suspect, or surface a missed thread. The NPC then responds to the combined exchange.

The partner earns XP from her interjections, weighted by alignment match and outcome. This is her primary action history — what feeds specialization generation for her roots.

**Combination unlocks:** When player and partner have both grown enough in complementary skills, certain approaches become available that neither could execute alone. These are surfaced in `/me`.

---

## Section 5: Achievement Surfacing

Skill growth is invisible during normal play. Three moments surface it:

1. **New specialization unlocked:** After the conversation ends, the partner delivers a single ambient line in her voice — not a system message. Something like *"You're getting good at that."* The specialization name and description are stored but not announced explicitly.

2. **Combination unlock:** Same treatment — one quiet partner line. The combination appears in `/me`.

3. **`/me` command:** A new Skills section shows:
   - Player's root levels and XP progress (no bar — just level number)
   - Each specialization earned under its root, described in the world's voice
   - Partner's skills and specializations alongside, clearly distinguished
   - Active combination unlocks

## Section 6: Storage

### New Tables

```sql
CREATE TABLE player_skills (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    owner TEXT NOT NULL,         -- 'player' or 'partner'
    root TEXT NOT NULL,          -- 'authority', 'streetwise', 'empathy', 'cunning'
    level INTEGER DEFAULT 1,
    xp INTEGER DEFAULT 0
);

CREATE TABLE player_specializations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    owner TEXT NOT NULL,
    root TEXT NOT NULL,
    name TEXT NOT NULL,
    description TEXT NOT NULL,
    unlocked_at_level INTEGER NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE skill_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    owner TEXT NOT NULL,
    root TEXT NOT NULL,
    xp_awarded INTEGER NOT NULL,
    reason TEXT,
    case_id INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (case_id) REFERENCES cases(id)
);
```

`skill_events` is the context source for specialization generation — the LLM receives the last N events for a root when generating a new specialization name and description.

### Alignment Axis Change

Remove clamping on `law_chaos` and `good_evil` in `update_player_alignment`. Both axes become unbounded integers reflecting cumulative choices.

## Out of Scope

- Player-facing XP numbers (no progress bars or numeric display)
- Skill-gated hard locks on any action
- Manual skill assignment or respec
- Skill decay over time
